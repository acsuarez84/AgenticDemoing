How the two systems work
LEFT — Text Position: Each word carries a number that is its actual byte location in the original 19th-century source text. The word "silk" first appears at byte 46 in Gilroy's 1845 manuscript. "Arachne" sits at byte 16,383. Words never written in the source texts — like "erased" or "silenced" — carry the value 0. These positions feed the algorithm that shapes each textile pattern: the number determines ring spacing, spiral tightness, wave frequency, or ray count.
RIGHT — ASCII Code: ASCII (American Standard Code for Information Interchange) assigns every letter a fixed number: A=65, B=66, a=97, b=98, and so on. The algorithm sums all character codes in a word — "silk" becomes s(115)+i(105)+l(108)+k(107)=435. Unlike text positions, these values are universal: "silk" always equals 435 regardless of which book it appears in. The same word produces the same number everywhere.
The difference matters: Text positions are contextual — the same word in a different book produces a different pattern. ASCII sums are abstract — the word's internal spelling alone determines the shape. When both systems generate simultaneously from the same rhetoric, you see how context versus abstraction weaves different textiles from identical language.
